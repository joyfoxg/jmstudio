# 🧪 Joy Markdown Studio - 사용자 정의 수식/공식 DB 구축 가이드

Joy Markdown Studio는 사용자만의 고유한 수식, 물리 공식, 화학 반응식 등을 등록하여 에디터 사이드바에서 원클릭으로 즉시 삽입하고 검색할 수 있는 **사용자 정의 외부 DB 기능**을 제공합니다.

이 문서를 따라 자신만의 `math_db.json` 파일을 생성하고 등록해 보세요.

---

## 📁 1. 파일 이름 및 배치 경로

프로그램은 시작 시 특정 위치에 사용자 정의 DB 파일이 있는지 자동으로 감지하고 로드합니다.

* **파일명**: 반드시 **`math_db.json`**으로 작성해야 합니다.
* **배치 경로**:
  * **컴파일된 실행 파일(`JoyMarkdownStudio.exe`) 사용 시**: 실행 파일(`.exe`)과 **동일한 폴더(경로)**에 배치합니다.
  * **소스 코드 개발 환경(`python main.py` 등) 실행 시**: 프로젝트 최상위 루트 디렉토리(프로젝트 폴더 바로 아래)에 배치합니다.

> [!NOTE]
> 실행 파일 경로에 외부 `math_db.json` 파일이 존재할 경우, 프로그램 내부의 기본 수식 데이터베이스를 대체하여 사용자가 작성한 커스텀 DB 목록으로 사이드바 헬퍼가 전환됩니다.

## 📊 2. JSON 데이터 구조 (Schema)

`math_db.json` 파일은 아래와 같은 형식의 JSON 배열 객체로 구성됩니다.

```json
[
  {
    "category": "분류코드(영문)",
    "name": "수식/공식의 이름 (한글/영문)",
    "latex": "LaTeX 수식 코드 내용",
    "keywords": ["검색어1", "검색어2", "keyword3"]
  }
]
```

### 🔑 각 필드 상세 설명
1. **`category` (필수)**: 수식이 소속될 카테고리 탭을 구분합니다.
   * 기본 지원 카테고리 코드: `math` (수학), `physics` (물리), `bio` (생명/화학), `cs` (컴퓨터공학), `ee` (전자공학)
   * *참고*: 다른 영문 코드를 입력해도 카테고리 탭에 정상 분류되어 표시됩니다.
2. **`name` (필수)**: 사이드바 리스트 및 마우스 오버 툴팁에 표시될 수식의 이름입니다.
3. **`latex` (필수)**: 실제 에디터 창에 삽입될 **LaTeX 형식의 수식**입니다. (자세한 작성 요령은 3절 참고)
4. **`keywords` (필수)**: 사이드바 검색창에서 검색할 때 매칭될 단어들의 배열입니다. 한글, 영문 대소문자 모두 등록 가능합니다.

## ⚠️ 3. JSON에서 LaTeX 작성 시 중요 주의사항 (백슬래시 이스케이프)

JSON 파일은 텍스트 규격 상 **역슬래시(백슬래시, `\`)**를 이스케이프(Escape) 문자로 인식합니다.
따라서, LaTeX 문법에 들어가는 모든 백슬래시(`\`)는 **두 번(`\\`) 연속으로 작성**해야 정상적으로 렌더링되고 에디터에 삽입됩니다.

### 💡 이스케이프 작성 예시
* **일반 LaTeX**: `\int_{a}^{b} f(x) \, dx`
* **JSON 등록 시**: `\\int_{a}^{b} f(x) \\, dx` (백슬래시를 모두 `\\`로 변경)

* **일반 LaTeX**: `\frac{1}{2}`
* **JSON 등록 시**: `\\frac{1}{2}`

## 🔁 5. 적용 및 반영 확인 방법

1. 위 예시를 바탕으로 만든 `math_db.json` 파일을 실행 파일 폴더에 넣습니다.
2. **Joy Markdown Studio** 프로그램을 실행(또는 이미 켜져 있다면 재시작)합니다.
3. 좌측 탭 중 **"수식 입력기"**를 선택합니다.
4. 카테고리 탭이나 수식 목록이 사용자가 정의한 파일 내용으로 바뀐 것을 확인합니다.
5. 검색창에 `keywords`에 등록한 단어(예: "광합성" 혹은 "pythagoras")를 입력하여 원하는 수식이 필터링되는지 테스트합니다.

## 🔄 6. 내장 수식 DB 작동 방식 및 병합(Merge) 방법

### ⚠️ 내부 파일 덮어쓰기(Override) 특성
* 컴파일 시 기본 내장 수식 파일(`math_db.json`)이 프로그램(`.exe`) 내부에 패키징되어 포함되지만, 실행 시 프로그램 실행 경로에 **외부 `math_db.json` 파일이 발견되면 내부의 기본 수식 목록은 완전히 무시(은폐)**되고 외부 파일의 내용만 검색창 및 목록에 노출됩니다.
* 단, 사이드바 화면상에 고정 디자인되어 있는 **기초 기호 버튼들(그리스 문자, 기본 사칙연산 기호 등)**은 무시되지 않고 정상 유지됩니다.

### 💡 내 수식과 기본 공식 목록을 같이 유지하는 방법 (병합)
기존 내장 공식(약 180개)을 그대로 유지하면서 나만의 공식을 추가하고 싶을 때는 아래와 같이 기본 파일을 템플릿으로 가져와 병합하면 됩니다.

1. 프로젝트 소스 폴더 내의 기본 수식 데이터 파일인 [frontend/static/data/math_db.json](file:///e:/jm_studio/frontend/static/data/math_db.json) 파일을 복사합니다.
2. 컴파일된 실행 파일(`.exe`)이 있는 폴더 경로에 `math_db.json`이라는 이름으로 저장(붙여넣기)합니다.
3. 복사한 외부 `math_db.json` 파일을 메모장이나 코드 에디터로 엽니다.
4. 파일의 맨 마지막 줄 부근 대괄호 닫기(`]`) 바로 앞의 수식 중괄호(`}`) 뒤에 **쉼표(`,`)**를 추가하고, 본인의 커스텀 수식 객체들을 추가합니다.
   ```json
       // (기존의 마지막 공식 항목 끝부분 예시)
       "keywords": ["nyquist", "sampling", "rate", "frequency", "나퀴스트", "샘플링"]
     }, // <--- 쉼표 추가!
     { // <--- 여기서부터 나만의 커스텀 수식 객체 작성 시작
       "category": "math",
       "name": "나만의 공식",
       "latex": "y = ax + b",
       "keywords": ["일차함수", "기울기"]
     }
   ]
   ```
5. 저장 후 프로그램을 실행하면 기존 내장 수식들과 내가 추가한 커스텀 수식들이 혼합되어 모두 정상적으로 조회 및 검색됩니다.
